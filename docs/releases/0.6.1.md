# codex-runtime 0.6.1

## Summary

- added `ClientConfig` launch controls for per-runtime `env`, spawn `cwd`, and extra `app-server` args
- added `Session::ask_stream(...)` and typed turn-scoped stream events on the canonical runtime bridge
- expanded typed live event extraction for agent message deltas and common turn terminal notifications
- made unfinished prompt streams fail-safe by sending a best-effort interrupt on drop

## Validation

- `./scripts/release_preflight.sh`
- `cargo publish -p codex-runtime --dry-run --allow-dirty`

## Highlights

- integrators can now configure Codex child process launch state without global environment mutation
- session streaming no longer requires direct global `subscribe_live()` filtering boilerplate for common cases
- prompt terminal resolution is now shared through a smaller data-first path across streaming and non-streaming flows
